TRON API method that generates trigger input parameters for shielded TRC20 contract calls. This method converts shielded TRC20 transaction parameters into formatted input data that can be used to trigger shielded contract operations.

<Warning>
This endpoint requires valid shielded TRC20 parameters generated from prior shielded transaction preparation. All hex values must be provided **without the 0x prefix**. You must specify the `parameter_type` field with one of these values: "mint", "burn", or "transfer". The parameters must include valid zero-knowledge proofs, binding signature, and message hash from the shielded transaction protocol.
</Warning>

## Parameters

* `shielded_TRC20_Parameters` тАФ object containing shielded transaction parameters
  * `spend_description` тАФ array of spend descriptions for transaction inputs
    * `value_commitment` тАФ value commitment (hex without 0x prefix)
    * `anchor` тАФ Merkle root of note commitment tree (hex without 0x prefix)
    * `nullifier` тАФ nullifier to prevent double spending (hex without 0x prefix)
    * `rk` тАФ public key for spend authorization signature verification (hex without 0x prefix)
    * `zkproof` тАФ zero-knowledge proof data (hex without 0x prefix)
  * `receive_description` тАФ array of receive descriptions for transaction outputs (required for transfer operations)
    * `value_commitment` тАФ value commitment (hex without 0x prefix)
    * `note_commitment` тАФ note commitment (hex without 0x prefix)
    * `epk` тАФ ephemeral public key (hex without 0x prefix)
    * `c_enc` тАФ encrypted note ciphertext (hex without 0x prefix)
    * `c_out` тАФ output ciphertext for auditing (hex without 0x prefix)
    * `zkproof` тАФ zero-knowledge proof data (hex without 0x prefix)
  * `binding_signature` тАФ binding signature for the transaction (hex without 0x prefix)
  * `message_hash` тАФ hash of the transaction message (hex without 0x prefix)
  * `parameter_type` тАФ type of shielded operation: "mint", "burn", or "transfer"
* `spend_authority_signature` тАФ array of spend authority signatures (one for each spend description)
  * `value` тАФ spend authorization signature (hex without 0x prefix)

## Response

* `trigger_contract_input` тАФ the formatted input data for triggering the shielded TRC20 contract
